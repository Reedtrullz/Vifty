# Vifty v1.3.0 Operator Clarity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Vifty's macOS interface state-first, predictable, accessible, and easier to operate across the main window, curve editor, menu-bar popover, Settings, and compact layouts without weakening fan-control safety.

**Architecture:** Introduce pure fan-control draft and control-session presentation types first. Keep AppModel as the hardware/control authority, but make it expose explicit draft/apply/session state to focused SwiftUI views. Split the oversized main view into scene, readiness, fan-control, telemetry, row, and Settings components; keep chart geometry and presentation pure so the right-fan visibility behavior is deterministic and testable.

**Tech Stack:** Swift 6, Swift Package Manager, SwiftUI, AppKit, macOS 15+, XCTest, existing make verify/make verify-full gates, bundled .app launch validation, and Build macOS Apps guidance for scenes, view boundaries, windows, adaptive materials, and runtime inspection.

## Scope Decision

This is one coordinated plan because every visible surface depends on the same control-session truth: readiness and ownership determine the primary action, the draft/apply model determines editor behavior, and the same presentation is consumed by the main window, menu bar, Settings entry points, fan rows, and accessibility labels. Each task below has a focused interface and an independent test cycle; Task 1 is the only prerequisite for the later visual tasks.

The product theme is Operator Clarity. The separate v1.3.0 release gate remains hardware-confidence evidence: UI work does not promote MacBookPro18 or any other model family to validated support without the existing read-only readiness, manual smoke, Auto-restore, and reviewed artifact evidence requirements.

struct FanCurveDraft: Equatable {

    var rampRPM: Double
    var highTemperature: Double

    var mode: ModeSelection
    var manualRunLimit: ManualRunLimit
    var fixedRPM: Double
    var fixedFanTargets: [FixedFanTarget]
    var usePerFanFixedRPM: Bool
    var curve: FanCurveDraft
    var selectedSensorID: String?
    var usePerFanOverrides: Bool
    var fanOverrides: [FanCurveOverride]

    func isDirty(comparedTo applied: FanControlDraft) -> Bool {
        self != applied
    }
}

enum FanControlApplyResult: Equatable {
    case applied
    case blocked(reason: String)
    case failed(message: String)
}

enum FanControlApplyState: Equatable {
    case applied
    case pending
    case applying
    case blocked(reason: String)
    case failed(message: String)
}
